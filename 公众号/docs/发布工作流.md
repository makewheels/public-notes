# 公众号项目结构

```
公众号/
├── .env                    # API 密钥（不提交 Git）
├── .gitignore              # Git 忽略规则
├── .venv/                  # Python 虚拟环境
├── scripts/                # 脚本
│   └── upload_draft.py    # 草稿上传脚本
├── docs/                   # 文档
│   ├── 写作指南.md
│   ├── 发布工作流.md
│   └── 封面生成指南.md
├── articles/               # 文章
│   └── YYYY-MM-DD-标题/
│       ├── draft/         # 草稿
│       │   └── article.md
│       ├── published/     # 发布后（空）
│       └── cover.svg      # 封面
└── README.md               # 项目说明
```

## 配置

`.env` 文件格式：
```
WECHAT_APPID=你的AppID
WECHAT_SECRET=你的AppSecret
```

⚠️ `.env` 文件不要提交到 Git！

# 发布工作流

每次发布文章按这个流程执行。

## 📋 流程概览

```
1. 写文章 → articles/draft/YYYY-MM-DD-标题/draft/article.md
2. 生成封面 → cover.svg
3. 上传草稿 → 脚本自动处理
4. 定时发布 → 用户后台操作
```

## 第一步：写文章

```
articles/YYYY-MM-DD-标题/draft/article.md
```

- 按写作指南完成
- 和用户确认最终版本

### 插图规则

**文章必须有图，不能纯文字。**

| 文章长度 | 插图数量 |
|---------|---------|
| < 2000 字 | 1 张 |
| 2000～4000 字 | 2 张 |
| > 4000 字 | 3 张 |

**插图位置建议：**
- 开头部分（吸引注意）
- 中间关键论点处（辅助理解）
- 总结之前（加深印象）

---

## 第二步：生成封面

**设计要点**：
- 尺寸：900×383（2.35:1）
- 标题大且醒目（字号≥60）
- 日期用**发布日期**
- 以文字为主
- 深色背景 + 亮色文字

**只保存 SVG，不要保存 PNG！**

---

## 第三步：上传草稿

### 固定流程（每次都按这个走）

```bash
cd ~/Documents/public-notes/公众号
.venv/bin/python scripts/upload_draft.py articles/YYYY-MM-DD-标题 <标题> <摘要>
```

示例：
```bash
.venv/bin/python scripts/upload_draft.py articles/2026-04-02-ai-impact-gap \
  "程序员慌了，CAD工程师为什么不慌？" \
  "为什么同样是数字化工作者..."
```

### 脚本功能

1. 从 `.env` 读取 API 配置
2. 获取 Access Token
3. SVG 转 PNG（调用 `rsvg-convert`）
4. 上传封面到永久素材
5. Markdown 转 HTML（用 `markdown` 库）
6. 创建草稿
7. 删除临时 PNG

### 依赖

```bash
# Python 依赖（已安装在 .venv）
pip install markdown

# 系统依赖
brew install librsvg  # macOS
```
