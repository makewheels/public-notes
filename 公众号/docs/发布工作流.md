# 公众号项目结构

```
公众号/
├── .env                    # API 密钥（不提交 Git）
├── .gitignore              # Git 忽略规则
├── .venv/                  # Python 虚拟环境
├── scripts/                # 脚本
│   ├── upload_draft.py    # 草稿上传脚本
│   └── upload_image.py    # 图片上传脚本
├── docs/                   # 文档
│   ├── 写作指南.md
│   ├── 发布工作流.md
│   └── 封面生成指南.md
├── articles/               # 文章
│   └── YYYY-MM-DD-标题/
│       ├── draft/         # 草稿
│       │   └── article.md
│       ├── published/     # 发布后（空）
│       ├── cover.svg      # 封面
│       └── illustration*.svg  # 插图（本地SVG，上传时自动处理）
└── README.md               # 项目说明
```

## 配置

`.env` 文件格式：
```
WECHAT_APPID=你的AppID
WECHAT_SECRET=你的AppSecret
```

⚠️ `.env` 文件**绝对不要**提交到 Git！`.gitignore` 中已排除。

# 发布工作流

每次发布文章，**AI必须一次性完成全部步骤**，不要分步手动做。

## 📋 完整流程（一次执行完毕）

```
1. 写文章 → draft/article.md
2. 生成插图 → illustration*.svg → PNG → 上传素材库 → 插入URL
3. 生成封面 → cover.svg
4. 上传草稿 → 脚本自动处理
5. 更新文章列表 → 状态改为"已上传草稿"
6. 通知用户 → 告知完成
```

---

## 第一步：写文章

```
articles/YYYY-MM-DD-标题/draft/article.md
```

- 按写作指南完成
- 标题确认后开始后续流程

### 插图规则

**文章必须有图，不能纯文字。**

| 文章长度 | 插图数量 |
|---------|---------|
| < 2000 字 | 1 张 |
| 2000～4000 字 | 2 张 |
| > 4000 字 | 3 张 |

**插图位置建议：**
- 开头部分（吸引注意）
- 中间关键论点处（辅助理解）
- 总结之前（加深印象）

---

## 第二步：生成插图

根据文章内容设计插图，保存为 `illustration1.svg`、`illustration2.svg` 等，放在文章目录下（与 `draft/` 同级）。

设计要点：
- 尺寸：800×400 或 800×450
- 深色背景 + 亮色文字/线条
- 图表用简洁风格（饼图、柱状图、对比图）
- 文字用中文，字号≥16

### ⚠️ 图片引用规则（重要！）

**Markdown 源文件中只能引用本地 SVG 路径，绝对不能写微信图片 URL。**

正确写法：
```markdown
![](../illustration1.svg)
![](../illustration2.svg)
```

❌ **禁止**在 article.md 中写微信 URL：
```markdown
![](http://mmbiz.qpic.cn/sz_mmbiz_png/xxx/0?wx_fmt=png)
```

**上传草稿脚本（upload_draft.py）会自动完成以下步骤：**
1. 扫描 markdown 中的本地 SVG 引用
2. SVG → PNG 转换（rsvg-convert）
3. 上传 PNG 到微信素材库
4. 在生成的 HTML 中替换为微信图片 URL
5. 清理临时 PNG 文件

这样 markdown 源文件始终保持干净，微信 URL 只存在于上传后的 HTML 中。

---

## 第三步：生成封面

**设计要点**：
- 尺寸：900×383（2.35:1）
- 标题大且醒目（字号≥60）
- 日期用**发布日期**
- 以文字为主
- 深色背景 + 亮色文字

保存为 `cover.svg`，**不要保存 PNG**（脚本会自动转换）。

---

## 第四步：上传草稿

```bash
cd ~/Documents/public-notes/公众号
.venv/bin/python scripts/upload_draft.py articles/YYYY-MM-DD-标题 <标题> <摘要>
```

### 草稿去重规则（重要！）

- **上传脚本已内置去重逻辑**：如果公众号后台已有同标题的草稿，会自动删除旧的再创建新的
- **关键：修改文章后重新上传时，必须使用相同的标题**，否则去重无效，会创建多篇草稿
- **绝对禁止**：每次修改都换一个新标题，这样后台会出现多篇草稿

### 脚本功能

1. 从 `.env` 读取 API 配置
2. 获取 Access Token
3. 检查重复草稿并删除旧的
4. SVG 转 PNG（封面）
5. 上传封面到永久素材
6. **扫描 markdown 中的本地 SVG 引用 → 转PNG → 上传 → 获取URL**
7. Markdown 转 HTML（**在 HTML 中替换为微信图片 URL**）
8. 创建草稿
9. 删除临时 PNG

---

## 第五步：更新文章列表

编辑 `docs/文章列表.md`，将状态改为：
- `📤 已上传草稿`

---

## 第六步：通知用户

发送消息：
```
草稿已上传成功！
- 标题：xxx
- 封面：已生成
- 插图：x 张

请去公众号后台查看草稿，设置定时发布。
```

---

## ✅ 检查清单（AI自检）

- [ ] 文章已写好并保存
- [ ] **检查：标题和上次上传保持一致**（不要改标题，否则会产生多篇草稿）
- [ ] 插图 SVG 已生成，放在文章目录下
- [ ] **检查：article.md 中引用的是本地 SVG 路径（如 `![](../illustration1.svg)`），不是微信 URL**
- [ ] 封面 SVG 已生成
- [ ] 草稿上传脚本执行成功
- [ ] 文章列表状态已更新
- [ ] 已通知用户
- [ ] **敏感信息检查**：确认 `.env` 没有提交到 Git，代码中没有硬编码的 AppSecret、API Key、密码

---

## 定时发布

用户自己去公众号后台：
1. 查看草稿
2. 设置定时发布

## 用户说"可以了" = 最终确认

当用户说"可以了"，意味着文章已审核通过，AI **必须立即执行**：

```bash
git add -A && git commit && git push origin master
```

**不要问用户，直接提交。**
