# 公众号项目结构

```
公众号/
├── .env                    # API 密钥（不提交 Git）
├── .gitignore              # Git 忽略规则
├── .venv/                  # Python 虚拟环境
├── scripts/                # 脚本
│   ├── upload_draft.py    # 草稿上传脚本
│   └── upload_image.py    # 图片上传脚本
├── docs/                   # 文档
│   ├── 写作指南.md
│   ├── 发布工作流.md
│   └── 封面生成指南.md
├── articles/               # 文章
│   └── YYYY-MM-DD-标题/
│       ├── draft/         # 草稿
│       │   └── article.md
│       ├── published/     # 发布后（空）
│       ├── cover.svg      # 封面
│       └── illustration*.png  # 插图（已上传）
└── README.md               # 项目说明
```

## 配置

`.env` 文件格式：
```
WECHAT_APPID=你的AppID
WECHAT_SECRET=你的AppSecret
```

⚠️ `.env` 文件不要提交到 Git！

# 发布工作流

每次发布文章，**AI必须一次性完成全部步骤**，不要分步手动做。

## 📋 完整流程（一次执行完毕）

```
1. 写文章 → draft/article.md
2. 生成插图 → illustration*.svg → PNG → 上传素材库 → 插入URL
3. 生成封面 → cover.svg
4. 上传草稿 → 脚本自动处理
5. 更新文章列表 → 状态改为"已上传草稿"
6. 通知用户 → 告知完成
```

---

## 第一步：写文章

```
articles/YYYY-MM-DD-标题/draft/article.md
```

- 按写作指南完成
- 标题确认后开始后续流程

### 插图规则

**文章必须有图，不能纯文字。**

| 文章长度 | 插图数量 |
|---------|---------|
| < 2000 字 | 1 张 |
| 2000～4000 字 | 2 张 |
| > 4000 字 | 3 张 |

**插图位置建议：**
- 开头部分（吸引注意）
- 中间关键论点处（辅助理解）
- 总结之前（加深印象）

---

## 第二步：生成并上传插图

### 2.1 创建插图SVG

根据文章内容设计插图，保存为 `illustration1.svg`、`illustration2.svg` 等。

设计要点：
- 尺寸：800×400 或 800×450
- 深色背景 + 亮色文字/线条
- 图表用简洁风格（饼图、柱状图、对比图）
- 文字用中文，字号≥16

### 2.2 转换为PNG

```bash
cd ~/Documents/public-notes/公众号/articles/YYYY-MM-DD-标题
rsvg-convert -f png -o illustration1.png illustration1.svg
rsvg-convert -f png -o illustration2.png illustration2.svg
```

### 2.3 上传到素材库

```bash
cd ~/Documents/public-notes/公众号
.venv/bin/python scripts/upload_image.py articles/YYYY-MM-DD-标题/illustration1.png
.venv/bin/python scripts/upload_image.py articles/YYYY-MM-DD-标题/illustration2.png
```

脚本返回 JSON，包含 `url` 字段，格式如：
```
{"media_id": "...", "url": "http://mmbiz.qpic.cn/sz_mmbiz_png/xxx/0?wx_fmt=png"}
```

### 2.4 插入URL到文章

将返回的 `url` 插入到文章 markdown 中：

```markdown
![](http://mmbiz.qpic.cn/sz_mmbiz_png/xxx/0?wx_fmt=png)
```

位置放在相关段落之后。

---

## 第三步：生成封面

**设计要点**：
- 尺寸：900×383（2.35:1）
- 标题大且醒目（字号≥60）
- 日期用**发布日期**
- 以文字为主
- 深色背景 + 亮色文字

保存为 `cover.svg`，**不要保存 PNG**（脚本会自动转换）。

---

## 第四步：上传草稿

```bash
cd ~/Documents/public-notes/公众号
.venv/bin/python scripts/upload_draft.py articles/YYYY-MM-DD-标题 <标题> <摘要>
```

### 草稿去重规则（重要！）

- **上传脚本已内置去重逻辑**：如果公众号后台已有同标题的草稿，会自动删除旧的再创建新的
- **关键：修改文章后重新上传时，必须使用相同的标题**，否则去重无效，会创建多篇草稿
- **绝对禁止**：每次修改都换一个新标题，这样后台会出现多篇草稿

示例：
```bash
.venv/bin/python scripts/upload_draft.py articles/2026-04-04-harness-engineering \
  "模型和Harness，谁更重要？" \
  "2026年AI圈最火的争论：模型和Harness谁更重要？真相是分工不同..."
```

### 脚本功能

1. 从 `.env` 读取 API 配置
2. 获取 Access Token
3. 检查重复草稿并删除旧的
4. SVG 转 PNG（封面）
5. 上传封面到永久素材
6. Markdown 转 HTML
7. 创建草稿
8. 删除临时 PNG

---

## 第五步：更新文章列表

编辑 `docs/文章列表.md`，将状态改为：
- `📤 已上传草稿`

```
- 插图：x 张

请去公众号后台查看草稿，设置定时发布。
```
