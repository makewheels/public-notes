你让 AI 帮你写了一段 2000 字的周报总结，你觉得这算一次调用，还是 100 次调用？

答案可能会让你意外——可能算 300 次，甚至更多。

这背后的核心概念叫 **Token**。如果你用 ChatGPT 以外的 AI 工具，或者未来打算自己接 API 做点什么，不理解这个概念，你就根本不知道钱是怎么花掉的。

https://developers.openai.com/api/docs/concepts#tokens

## 一、Token 不是字，不是词，是"词元"

OpenAI 的官方定义：**Tokens represent commonly occurring sequences of characters.**

Token 是文本中反复出现的"字符组合单位"。不是一句话，不是一个词，甚至不是一个字。

举个例子你就明白了：

- 英文短词 `the` → 1 个 Token
- 英文长词 `tokenization` → 拆成 `token` + `ization`，2 个 Token
- 中文 "你好" → 大约 2 个 Token
- 中文 "人工智能" → 可能 2-3 个 Token，取决于怎么切分

中文 AI 业界的标准翻译是：**词元**。

为什么中文比英文"费"Token？因为英文有天然分词（空格），而中文是连续的文字流，AI 需要用 BPE（字节对编码）算法一个字一个字地去拆分。所以同样内容的文本，翻译成中文后 Token 数通常会比英文多 20%-40%。

## 二、一次对话，Token 是怎么花掉的？

很多人以为：我问一个问题，AI 回答一句，这就是一次调用。

实际上你消耗了两种东西：

- **输入 Token（Input Tokens）**：你发过去的全部内容
- **输出 Token（Output Tokens）**：AI 生成的回复

还有一个隐藏的关键点：**每次对话不是只发当前这句话**。

如果你用 ChatGPT 的对话模式，或者用 Claude 的上下文对话，第 10 轮提问时，AI 收到的是前 9 轮的完整对话历史 + 你的第 10 个问题。这意味着 Token 消耗会随着对话轮数滚雪球式增长。

这就引出一个指标：**Context Window（上下文窗口）**。

| 模型 | 上下文窗口 | 大约能装多少字 |
|------|-----------|----------------|
| GPT-5.4 | 50 万 Token | ~100 万汉字 |
| Qwen 3.6 Plus | 100 万 Token | ~200 万汉字 |
| Grok 4.20 | 200 万 Token | ~400 万汉字 |

你可以把它理解为 AI 的"工作记忆容量"。不是越大越好——越大越贵，但确实能处理更长的文档。

## 三、价格是怎么定的？

所有 AI API 的统一计费单位：**每 100 万 Token（per million tokens）**。

目前主流模型的定价：

| 模型 | 输入 | 输出 | 缓存输入 |
|------|------|------|---------|
| GPT-5.4 | $2.50 | $15.00 | $0.25 |
| GPT-5.4-mini | $0.75 | $4.50 | $0.075 |
| Claude Opus 4.6 | $5.00 | $25.00 | $0.50 |
| Claude Sonnet 4.6 | $3.00 | $15.00 | $0.30 |
| Qwen 3.6 Plus | $2.00 | $12.00 | $0.20 |

三个规律值得注意：

**第一，输出比输入贵 5-7 倍。** 输出是模型真正的"思考"和"计算"部分。

**第二，缓存输入极便宜，只有正常输入的 10%。** 这是目前省钱的"暗门"，后面单独讲。

**第三，模型之间差价能到几十倍。** 同一个任务，GPT-5.4-nano 的输出价格 $1.25/M，而 GPT-5.4-pro 高达 $270/M，差 216 倍。但日常写邮件、做总结这种简单任务，nano 和 pro 的差别可能根本感觉不到。

## 四、什么是 Token 缓存？为什么能省一半以上？

想象你去餐厅点外卖。你每次都点不同的菜，厨师每次都要从头做。但如果你连续几天点同一道菜（只是加了一点点新的配菜），聪明的厨师会把之前做好的主菜存进冰箱，只重新做那个小配菜。

**Token 缓存就是这个原理。**

如果你的输入内容和之前很像——比如同一个项目代码库反复对话，90% 的代码文件没变——AI 供应商会把之前处理过的结果存起来。下次匹配到相同内容时只收 10% 的费用。

实际开发中，缓存命中率通常在 50%-90%。这意味着**同一对话的后续轮次，实际成本可能只有第一次的四分之一**。

## 五、免费模型到底"免费"在哪？

你可能在 OpenRouter 上看到过 Qwen 3.6 的 Free 版本。这到底是什么意思？

免费模型不是无限的。它有两层限制：

**请求频率限制：** 每分钟 20 次请求，每天 50 次请求。50 次是什么意思？你写一篇文章跟 AI 来回讨论十几轮，一天可能就用掉 3 次。50 次够你用十几天。

**模型选择限制：** 免费模型池里的模型不固定。今天可能是 Qwen，明天可能换成其他。而且免费模型通常有较低的速度优先级，高峰期可能要排队等。

结论：免费模型适合试玩和轻度使用，不适合生产环境或重度对话。

## 六、除了文字，AI 还在收什么钱？

AI 已经不只是聊天了。现在的 API 还能生成图片、视频、录音，甚至控制你的电脑。这些计费方式完全不同：

- **图片生成**（GPT Image）：按张收费，$0.03 到 $0.16 不等，取决于分辨率
- **视频生成**（Sora）：按秒计时，720p 每秒钟 $0.10，1080p 每秒 $0.70
- **音频转写**（Whisper）：按分钟计时，约 $0.006/分钟
- **Computer Use**（AI 控制电脑）：$12/小时
- **Deep Research**（自主调研）：输入 $10/M + 输出 $40/M，一次深度调研报告可能花掉几美元到十几美元

最离谱的可能是文件搜索：按 GB 计费，**$0.10/GB 每天，超过 1GB 免费额度后开始计费**。这意味着如果你上传了一个 10GB 的项目代码库，每天光存储就花 $0.90，一个月 $27。

## 七、普通用户怎么省钱？

如果你不是开发者，只是每天用用 ChatGPT、文心一言之类的产品，这些钱不需要你操心。但如果你开始用 API 或者自己接服务，这些技巧能帮你省不少：

**挑对模型，比省 Token 重要 100 倍。** 日常写邮件、做总结用 nano 级别就够了，用 pro 属于杀鸡用牛刀。同一个任务，差价能到 200 倍。

**用上下文缓存，别让 AI 反复"读"同样的背景材料。** 这是目前 AI 服务能省钱的最大的一个动作。

**缩短对话，别让历史太长。** 对话轮数越多，每一轮消耗越多。定期开新对话，比一直在一个对话里聊要便宜得多。

**中文更"贵"的真相。** 同样 5000 字的内容，写成中文比英文多消耗 20%-40% 的 Token。如果你在做多语言产品，用英文写 prompt 本身就能省钱。

## 八、算一笔实际账

假设你每天用 AI 辅助开发，平均发 100 次对话，每次带 5000 Token 上下文，AI 回复 3000 Token。

按 **Qwen 3.6 Plus**（输入 $2/M，输出 $12/M）算：
- 输入费用：0.5M × $2.00 = $1.00
- 输出费用：0.3M × $12.00 = $3.60
- **总计：$4.60/天，约 ¥33**

缓存命中率 70% 的情况下：
- 实际输入费用：0.5M × 70% × $0.20 + 0.5M × 30% × $2.00 = $0.07 + $0.30 = $0.37
- **实际总计：约 $4.00/天，约 ¥28**

按 **GPT-5.4**（输入 $2.50/M，输出 $15/M）算：
- **约 $5.80/天，约 ¥42**

个人用的话不贵。但 10 人团队，一个月就是 **¥8400 到 ¥12600**。

这就是为什么大公司都在自建 AI 集群、做私有部署——不是买不起 API，是用到一定规模，自建反而更便宜。

Token 是 AI 的计量单位。理解了这个概念，你才能真正看懂 AI 的账单。
