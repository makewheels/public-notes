# 公众号发布工作流

每次发布文章按这个流程执行。

---

## 📋 流程概览

```
1. 写文章 → draft/article.md
2. 生成封面 → cover.svg → cover.png（上传后删除）
3. 上传微信 → 创建草稿 → 通知用户
```

---

## ⚠️ 重要规则

**只能发草稿，不能发布！**

- 个人订阅号 API 只能创建草稿
- 用户自己去公众号后台点击"发布"
- 我的工作：创建草稿 → 通知用户

**一天只发一篇！**

- 今天发的文章：今天的日期
- 多篇文章排队，明天发下一篇
- 记录待发布文章列表

**只保存 Markdown，中间产物都删掉！**

- ✅ 保留：`draft/article.md`（Markdown 源文件）
- ✅ 保留：`cover.svg`（封面源文件）
- ❌ 删除：`cover.png`（上传微信后删除）
- ❌ 删除：HTML 内容（直接发给微信，不保存）
- ❌ 删除：其他中间文件

**Git 只提交：**
- Markdown 文章
- SVG 封面
- 指南文档

**不上传 Git：**
- PNG 图片
- HTML 内容
- 其他中间产物

---

## 第一步：写文章

```bash
~/Documents/public-notes/公众号/YYYY-MM-DD-标题/draft/article.md
```

- 按写作指南完成
- 和用户确认最终版本

---

## 第二步：生成封面

**设计要点**：
- 尺寸：900×383（2.35:1）
- 标题大且醒目（字号≥60）
- 日期用**当天日期**（明天的文章用明天的日期）
- 以文字为主
- 深色背景 + 亮色文字

**生成命令**：
```bash
rsvg-convert -o cover.png cover.svg
```

**上传后删除 PNG**：
```bash
rm cover.png
```

详细设计规范见：`封面生成指南.md`

---

## 第三步：上传微信

1. **获取 token**（从本地配置读取）
2. **上传封面** → 永久素材 → 记录 media_id
3. **上传后删除本地 PNG**
4. **创建草稿** → 通知用户去后台发布

**HTML 换行技巧**：
- 每个段落用 `<p style="margin-bottom: 1.5em;">内容</p>`
- 段落之间加 `<p><br></p>` 强制空行
- 微信可能会忽略部分 CSS，用 `<br><br>` 更保险

---

## 📁 目录结构

```
公众号/
├── YYYY-MM-DD-标题/
│   ├── draft/
│   │   └── article.md     # Markdown 源文件（唯一保留的文章文件）
│   └── cover.svg          # 封面源文件（唯一保留的图片文件）
├── 写作指南.md
├── 封面生成指南.md
└── 发布工作流.md
```

**注意**：
- PNG 上传微信后立即删除
- HTML 内容直接发给微信 API，不保存本地
- 只有 `.md` 和 `.svg` 提交到 Git
