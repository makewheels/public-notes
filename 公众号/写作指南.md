# 公众号文章写作指南

每次写文章前，先读这个文件。

---

## 公众号定位

**名称**：AI来了
**简介**：AI随想录

**目标人群**：程序员、关注AI发展的人

**内容方向**：AI趋势、工具、思考、学习

**核心价值**：不是搬运新闻，而是**有深度的思考和总结**。

---

## 核心原则

### 1. 不是复述，是创作

❌ 把用户说的整理一下发出去
✅ 理解观点，用自己的话重新表达，加入思考

**用户给的是素材，不是成品。**

### 2. 读者不是技术专家

❌ 直接用术语（LangGraph、RAG）
✅ 用比喻、故事、生活例子

### 3. 要有深度思考

❌ 信息搬运
✅ 自己的洞察、观点升华

---

## 写作流程

1. **理解素材** → 核心信息是什么？目标读者是谁？
2. **和用户确认** → 读者是谁？目的是什么？什么风格？
3. **构思切入点** → 问题、故事、焦虑、现象
4. **用比喻解释** → 技术概念转成生活语言
5. **检查逻辑** → 观点是否成立？论证是否通顺？
6. **观点升华** → 结尾让人记住一句话

---

## 逻辑检查清单

写完后自问：

- [ ] 观点是否站得住脚？（不要说反直觉但其实是常识）
- [ ] 论证是否通顺？前后是否矛盾？
- [ ] 每个段落是否都在推进主题？
- [ ] 结尾是否回应了开头的问题？

---

## 文章结构

```
切入点（共鸣） → 展开（比喻） → 核心观点 → 升华（一句话）
```

---

## 配图建议

**为什么需要图片？**

纯文字太干，有图片更生动。

**图片来源（任选）：**

1. **Markdown 占位**：用 `![描述](图片URL)` 格式，后续补充图片
2. **网络图片**：从 Unsplash、Pexels 等找免费图片
3. **自己截图**：工具界面、代码截图等

**注意**：微信公众号图片需要单独上传，不能直接用 URL。但 Markdown 里先标记，方便后续处理。

---

## 检查清单

- [ ] 不懂技术的人能看懂吗？
- [ ] 是创作不是复述？
- [ ] 有生活化比喻吗？
- [ ] 结尾有让人记住的一句话？
- [ ] 和用户确认过了吗？
- [ ] 排版有足够空行吗？（不要太密集）
- [ ] 有配图吗？（可选，但建议有）

## 经验教训

### 2026-03-29 第一次复盘

**问题**：复述用户内容、用术语、没确认就写
**解决**：把素材当原材料消化后再创作，用比喻，先确认再写

**好的做法**：用比喻解释、从焦虑切入、有观点升华
